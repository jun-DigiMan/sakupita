# サクピタ プレゼンスライド構成 手順書
## 5分間・5スライド構成

---

## スライド全体の方針

- **トーン**: 技術自慢ではなく「営業課題を自分で解決した」ストーリー
- **フォント**: 見出し大きめ・本文少なめ（1スライド1メッセージ）
- **色**: サクピタのメインカラー（紺 `#1a2744`）をアクセントに使用
- **デモ**: スライド4の後にライブデモを挟む構成も可

---

## 📑 Slide 1 / タイトル（0:00〜0:30）

**レイアウト**: タイトル中央・サブタイトル下・ロゴ右下

| 要素 | 内容 |
|------|------|
| メインタイトル | **サクピタ** |
| サブタイトル | 営業チームの商談予約を、ゼロコストで自動化する |
| 補足 | 開発者: 野口 純 ／ 2026年3月 |
| ビジュアル | サクピタのスクリーンショット or ロゴ |

---

## 📑 Slide 2 / 課題（0:30〜1:30）

**レイアウト**: 左に課題リスト・右にビフォー図

| 要素 | 内容 |
|------|------|
| 見出し | **商談調整、こんなに手間がかかっていた** |
| 課題① | Calendlyなど既存ツールは外部向け。チーム複数人の空き確認ができない |
| 課題② | 確認メール・スプシ記録・リマインダーをすべて手作業 |
| 課題③ | 外部SaaSはコストがかかる or 自社業務に合わない |
| キーメッセージ | → **1回の予約操作で全部終わらせたかった** |

---

## 📑 Slide 3 / ソリューション概要（1:30〜2:30）

**レイアウト**: 中央にアプリ画面・周囲に機能アイコン

| 要素 | 内容 |
|------|------|
| 見出し | **サクピタが解決すること** |
| 機能① | チームカレンダーをリアルタイム表示 → 空き確認が一瞬 |
| 機能② | スロットをクリック → 顧客情報入力 → 予約完了まで1画面 |
| 機能③ | 確認メール・Googleカレンダー登録・スプシ記録を自動実行 |
| 機能④ | リマインダーを2営業日前に自動送信（GAS連携） |
| ビジュアル | カレンダー画面のスクリーンショット |

---

## 📑 Slide 4 / こだわりポイント（2:30〜4:00）

**レイアウト**: 3カラム or 縦3段

| # | 見出し | 内容 |
|---|------|------|
| 1 | **コスト$0** | バックエンド不要。GitHub Pages + Google APIのみ。サーバー代・SaaS代ゼロ |
| 2 | **スマート入力** | 氏名を苗字辞書30,685件 × メールアドレスのローマ字で自動分割。電話番号は市外局番2,000件で自動ハイフン |
| 3 | **Zoom / Meet 選択制** | 担当者ごとにZoom PMI設定 or Google Meet自動発行。会議URLを手動コピー不要 |

**補足テキスト（小さめ）**:
- モバイル完全対応（1週/2週 × 30分/60分 の4パターン）
- リスケジュールリンクを担当者メールに自動挿入

---

## 📑 Slide 5 / まとめ・今後（4:00〜5:00）

**レイアウト**: 左に成果・右に今後

| 要素 | 内容 |
|------|------|
| 見出し | **開発して変わったこと・これから** |
| 成果① | 商談1件あたりの調整作業 → **ほぼゼロ**に |
| 成果② | 予約ログがスプシに自動蓄積 → 商談データが資産化 |
| 成果③ | 開発期間 約2週間・コード5,521行・コスト$0 |
| 今後① | ICSカレンダー招待対応（顧客のカレンダーに直接追加） |
| 今後② | 苗字辞書 → 30万件拡充 |
| 締め | **自分たちの課題は自分たちで解決できる** |

## スライド作成時の注意

1. 1スライドに文字を詰め込まない（箇条書き最大4行）
2. スクリーンショットは実際のサクピタ画面を使用（デモ感を出す）
3. Slide 4の後にライブデモを挟むと効果的
4. フォントサイズ: 見出し36pt以上・本文20pt以上
