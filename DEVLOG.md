# サクピタ 開発ログ

## プロダクト概要

**サクピタ**（Saku-Pita）は、営業チームが顧客との商談日程を効率よく調整・予約できるWebアプリ。
Googleカレンダーと連携し、チームメンバー全員の空き時間をリアルタイムで確認しながら、顧客情報の入力・確認メール送信・スプレッドシート記録までワンストップで完結する。

- **URL**: https://jun-digiman.github.io/sakupita/
- **リポジトリ**: https://github.com/jun-DigiMan/sakupita
- **技術スタック**: HTML / CSS / JavaScript（Vanilla JS・フレームワークなし）
- **外部API**: Google Calendar API / Gmail API / Google Sheets API / Google Drive API
- **認証**: Google Identity Services (GIS) OAuth 2.0
- **ホスティング**: GitHub Pages（無料・常時稼働・`git push`で自動反映）
- **コード規模**: app.js 4,177行 / style.css 1,009行 / index.html 335行

---

## 開発の経緯・目的

- 既存の日程調整ツール（Calendly等）は外部向けで、社内複数メンバーのカレンダーを一括確認しながら調整する機能がない
- 顧客への確認メール送信・スプレッドシート記録など、商談調整に必要な作業を1つのツールに集約したかった
- Googleアカウントだけで完結するシンプルな設計にこだわった
- 外部SaaSに依存せず、自社で永続的に運用できるツールを目指した
- サーバーレス（静的ファイルのみ）で運用コストゼロ

## 実装機能一覧

### 🔐 認証・セッション管理

**実装内容：**
- Google Identity Services (GIS) の `requestAccessToken` でOAuth 2.0認証
- アクセストークンを有効期限付きで `localStorage`（`sakupita_token`）に保存
- リロード時に `gapi.client.setToken()` でトークンを即時復元 → サインイン画面を経由しない
- トークン期限の55分前にサイレント自動更新タイマーをセット → タブを開いている限り再認証不要
- トークン切れ時は**再接続オーバーレイ**を表示。カレンダー表示を維持したまま復帰可能
- サインアウト時は全localStorageをクリア

**こだわり：**
- リロードのたびにサインイン画面が出る問題の根本解決に注力。原因は `requestAccessToken({ prompt: '' })` が失敗時にコールバックを呼ばない仕様で、最終的にトークン自体をlocalStorageに保存・復元する方式に落ち着いた
- **モバイルSafari対応**: Safariはユーザーアクションなしのポップアップをブロックする。トークン切れ時の自動 `requestAccessToken()` 呼び出しを廃止し、「接続する」ボタン経由（ユーザー操作起点）に変更

---

### 📅 カレンダー表示

**実装内容：**
- 月〜金の5列グリッド × 時間帯別スロット表示（週表示）
- 前週・次週ナビゲーション
- Google Calendar freebusy APIで全メンバーの予定を取得、空き/予約済みをリアルタイム表示
- **busyDataキャッシュ**（`sakupita_busy_cache`）: 週単位・2時間TTLでキャッシュ。同じ週の再表示はAPIなしで即表示
- 1分ごとの自動更新（バックグラウンドで最新化）
- 表示パターン: **1週/2週 × 30分/60分** の4パターン切り替え

**こだわり（当日ハイライトのデザイン試行）：**
- 青色 → 「ベース色の紺と被って見づらい」
- 緑色 → 「浮いて見える」
- 水色の丸 → 「土曜日カラーっぽい」
- **最終決定: 紺色の丸に白文字**。メインカラーを崩さず視認性が最も高い

---

### 🎨 ブランディング・デザイン

- オリジナルロゴを複数箇所に配置（ヘッダー・ファビコン・確認メール・予約完了画面）
- ロゴはCanvasで生成しBase64エンコード → 外部ファイル参照なしのインライン表示
- メインカラー: 紺色（`#1a3353`）で統一。清潔感・信頼感を重視
- 土曜=青系、日曜=赤系でカラー分け
- 日本の祝日をハードコードで保持、祝日スロットは視覚的に区別・予約不可

---

### 👥 メンバー管理

- メンバー追加: 名前 + Gmailアドレスで登録
- メンバー削除: 「削除」ボタンで即時削除
- 「連携済み」「未連携」バッジで状態を色分け表示
- メンバーごとにカレンダー表示用カラーを自動割り当て（重複なし）
- メンバー情報をlocalStorageに永続保存

### 📝 予約フォーム

**入力項目：**

| フィールド | 必須 | バリデーション |
|----------|------|------------|
| 会社名 | ✅ | 空欄チェック |
| 担当者名 | ✅ | 空欄チェック + 自動スペース補完 |
| メールアドレス | ✅ | 正規表現形式チェック |
| 電話番号 | ✅ | ハイフン形式チェック + 自動フォーマット |
| 部署 | ✅ | 空欄チェック |
| 役職 | ✅ | 空欄チェック |
| コメント | — | なし |

**こだわり：**
- フォームを開いた瞬間に会社名フィールドへ自動フォーカス
- エラー時は該当フィールドを赤枠ハイライト + 具体的なフォーマットをアラートで案内
- ボタンテキストをリアルタイム更新（「処理中...」「メール到達確認中...」）で処理状況を明示

### 🔗 会議URL（Google Meet / Zoom 選択制）

**実装内容：**
- 予約フォームで **Google Meet** または **Zoom** をボタンで選択
- Google Meet: Calendar API が自動生成（`conferenceDataVersion: 1`）
- Zoom（PMI設定済みメンバー）: config.js の招待URLをそのまま使用（パスワード付き完全URL）
- Zoom（PMI未設定メンバー）: カスタムURL入力欄を表示

**現在の設定：**

| メンバー | 設定 |
|--------|------|
| 野口純 | Zoom PMI設定済み（パスワード付き招待URL） |
| 松居和輝 | 未設定（カスタムURL入力に誘導） |

**こだわり：**
- Zoom APIのOAuth実装を不要にするため、PMIの固定URLをconfig.jsに直書きする方式を採用
- 招待URLはパスワード付き完全URLとして保存し、加工せずそのまま使用（パスワード欠落バグを防止）

### 🏷 日本語名前自動分割

**変換例：**
```
入力: 山田太郎
出力: 山田 太郎
```

**アルゴリズム：**
- **苗字辞書**（`JP_SURNAMES`）: **30,685件**収録（5文字姓〜1文字姓、沖縄系姓も収録）
- メールアドレスの `@` 以前をローマ字ヒントとして活用して精度向上
- 2パート形式（`kim-koichiro`）は苗字↔名前の**双方向スコアリング**で最良を選択
- 1パート形式は苗字として扱う（日本のビジネスメール慣習）
- 漢字1文字の頭文字マップ（`KANJI_INITIALS`）でスコア補正
- 名前入力後→メール入力後の**2段階で自動補正**（入力順序を問わない）

**こだわり：**
- `金光一郎` + `kim-koichiro@...` → 「金光 一郎」vs「金 光一郎」を正しく判別
- メールアドレスの各パートをローマ字換算した長さと、漢字文字数から期待されるローマ字長の範囲（`ROMAJI_RANGES`）を比較してスコアリング

### 📧 メール送信・バウンス検知

**実装内容：**
- 予約完了時に**顧客**と**担当営業**の両方へ確認メールを自動送信（Gmail API）
- HTMLメール形式（サクピタロゴ・会議URL・商談日時を含む）
- **バウンス自動検知フロー**:
  1. メール送信
  2. 「メール到達確認中...」と表示しながら1.5秒待機
  3. Gmail受信箱を `mailer-daemon` / `postmaster` で検索
  4. バウンス検知 → アラート表示 + カレンダーに⚠️記録

**こだわり：**
- バウンス確認を**別の try/catch に分離**。バウンス確認APIが失敗しても「送信失敗」アラートが出ないよう修正（誤報の根絶）

### ⏰ GASリマインダー（Google Apps Script）

- 商談日の **2営業日前**（土日祝除く）に自動メール送信
- 日本の祝日は Google Calendar API（`ja.japanese#holiday`）から取得
- 顧客・担当者の両方にHTMLメール送信
- トリガー: 毎日9:00〜10:00に `sendReminders()` を実行

**こだわり：**
- 当初は「2日前」（カレンダー日数）だったが「2営業日前」に変更
- 土日祝を除いた営業日カウントを `subtractBusinessDays()` で実装

## モバイル対応

### ヘッダー2行レイアウト

```
1行目: [サクピタロゴ]  [< 今週 日付範囲 >]
2行目: [30分][60分] [1週][2週] [メンバー] [サインアウト]
```

- `flex-wrap: wrap` + `justify-content: center` で両行を中央揃え
- ブランド・ナビを `flex: 0 0 auto` にしてauto幅化

### スロットボタン（4パターン全対応）

| 表示 | 列幅 | 有効幅 | アバター種 |
|------|------|------|---------|
| 30分×1週 | ~65px | ~55px | pill型（sm）|
| 60分×1週 | ~65px | ~55px | pill型（md）|
| 30分×2週 | ~33px | ~27px | 円型12px |
| 60分×2週 | ~33px | ~27px | 円型12px |

- `.time-slot { padding: 1px 1px }` で列幅を最大活用
- 2名表示: 12px + 1px gap + 12px = 25px ≤ 有効幅27px ✓

## インフラ・デプロイ

### Google Cloud Console 設定

- プロジェクト名: `project-scheduler`
- OAuth クライアントID: `サクピタ`
- 承認済みJS生成元: `http://localhost:8766` / `https://jun-digiman.github.io`
- 有効スコープ: Calendar / Gmail / Sheets / Drive

### GitHub Pages

- リポジトリ: `jun-DigiMan/sakupita`
- ブランチ: `main`
- `git push origin main` でGitHub Pagesに自動反映（1〜2分）
- ローカル開発: `python3 -m http.server 8766`

### config.js 設定項目

```javascript
CLIENT_ID       // Google Cloud OAuth クライアントID
API_KEY         // Google API キー
MEMBERS         // メンバー名・カレンダーID・色・ZoomPMI
WORKING_HOURS   // 稼働時間 9:00〜20:00
SLOT_MINUTES    // 30分刻み
LUNCH_HOUR      // 12:00〜13:00 ランチ除外
DEMO_MODE       // true: モックデータ / false: 本番
SHEET_ID        // スプレッドシートID（固定）
```
