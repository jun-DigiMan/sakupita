# サクピタ 開発ログ

## プロダクト概要

**サクピタ**（Saku-Pita）は、営業チームが顧客との商談日程を効率よく調整・予約できるWebアプリ。
Googleカレンダーと連携し、チームメンバー全員の空き時間をリアルタイムで確認しながら、顧客情報の入力・メール送信・スプレッドシート記録までワンストップで行える。

- **URL**: https://jun-digiman.github.io/sakupita/
- **技術スタック**: 純粋な HTML / CSS / JavaScript（フレームワークなし）
- **外部API**: Google Calendar API / Gmail API / Google Sheets API / Google Drive API
- **認証**: Google Identity Services (GIS) OAuth 2.0

---

## 開発の経緯・目的

- 既存の日程調整ツール（Calendly等）は外部向けで、社内の複数メンバーのカレンダーを一括で見て調整する機能がない
- 顧客への確認メール送信・スプレッドシートへの記録など、商談調整に必要な一連の作業を1つのツールに集約したかった
- Googleアカウントだけで完結するシンプルな設計にこだわった
- 外部サービスに依存しない自社ツールとして永続的に運用できる形を目指した

## 実装した機能一覧

### 🔐 認証・セッション管理

- **Googleサインイン**: Google Identity Services (GIS) の `requestAccessToken` を使用
- **トークンのlocalStorage保存**: アクセストークンを期限付きでlocalStorageに保存し、リロード時に即復元。毎回の再認証を不要にした
- **サイレント自動更新**: トークン期限の5分前にバックグラウンドで自動更新するタイマーを実装。タブを開いている限り再認証画面が出ない（実質24時間以上セッション維持可能）
- **再接続オーバーレイ**: 万が一トークンが切れた場合、カレンダー表示を維持しつつ「再接続」ボタンを表示。ページ遷移なしに復帰できる
- **サインアウト**: ヘッダー右端にサインアウトボタン（SVGアイコン）を設置。サインアウト時はlocalStorageを全クリアしてサインイン画面へ遷移

### 📅 カレンダー表示

- **週表示グリッド**: 月〜日の7列 × 時間帯別でスロットを表示
- **前週・次週ナビゲーション**
- **当日ハイライト（こだわり）**: 今日の日付を**紺色の丸に白文字**で表示。視認性を最大化するデザインにこだわった
  - 水色・緑・青と複数案を検討し、「浮いて見える」問題を解消する紺丸＋白文字に決定
  - 曜日名も紺色で統一し、全体のトーンを崩さない
- **土日のカラー分け**: 土曜=青系、日曜=赤系で直感的に判別可能
- **祝日反映（こだわり）**: 日本の祝日をハードコードで保持し、祝日スロットを視覚的に区別。祝日は予約不可にして誤予約を防止
- **busyDataキャッシュ**: Googleカレンダーの空き時間データを2時間TTLでlocalStorageにキャッシュ。同じ週のナビゲーション時はAPIを再呼び出しせず高速表示
- **予約後のリアルタイム反映**: 予約完了後、全メンバーのカレンダーグリッドを即時更新

### 🎨 ブランディング・デザイン（こだわり）

- **サクピタロゴの作成・配置**: オリジナルロゴをCanvasで生成し、ヘッダー・ファビコン・メール本文・予約完了画面など複数箇所に配置
  - ロゴは白抜き版と通常版の2種類を用意し、背景色に応じて使い分け
  - Base64エンコードして外部ファイル不要のインライン表示を実現
- **ナビゲーションバー**: 紺色ベースのヘッダーにロゴ・メンバー管理ボタン・サインアウトボタンを配置
  - メンバー管理（左）・サインアウト（右）の配置順もUXを考慮して決定
- **カラースキーム**: メインカラーに紺色（`--color-navy`）を採用。清潔感と信頼感を重視

### 👥 メンバー管理

- **メンバー追加・削除**: 名前とGmailアドレスで追加
- **連携状態バッジ**: 「連携済み」「未連携」を色分けバッジで表示
- **カラー自動割り当て**: メンバーごとにカレンダー上のカラーを自動割り当て。重複なし
- **URLからメンバー追加**: `calendar.app.google` のURLを貼り付けてメンバー追加を試みる（CORSプロキシ3種を順次試行）

### 📝 予約フォーム（こだわり）

**入力項目**（コメント以外すべて必須）:
| 項目 | バリデーション・UX |
|------|----------------|
| 会社名 | 必須・フォームオープン時に自動フォーカス |
| 担当者名 | 必須・タブ移動時にスペース自動補完 |
| メールアドレス | 必須・形式チェック（正規表現）|
| 電話番号 | 必須・ハイフン形式チェック・自動フォーマット |
| 部署 | 必須 |
| 役職 | 必須 |
| コメント | 任意 |

- **全フィールドにプレースホルダー**: グレー文字で入力例を常時表示（例: 「株式会社サクピタ」「山田 太郎」「03-1234-5678」）
- **フォームオープン時に会社名フィールドへ自動フォーカス**: 入力開始のストレスゼロ

### 📞 電話番号スマートフォーマット（こだわり）

タブ移動時に全角・ハイフンなし入力を自動整形する独自実装：

```
11桁 (090/080/070/050): XXX-XXXX-XXXX
10桁 2桁市外局番 (03/06): XX-XXXX-XXXX
10桁 3桁市外局番 (011等): XXX-XXX-XXXX
10桁 4桁市外局番 (0138等): XXXX-XX-XXXX
10桁 5桁市外局番 (01372等): XXXXX-X-XXXX
10桁 特番 (0120/0570/0800/0990): XXXX-XXX-XXX
```

- 全角数字・全角ハイフンを自動で半角変換
- **市外局番データベース（約200種類）** を搭載し、地域ごとに最適なハイフン位置を判定
- 5桁まで対応する最長一致アルゴリズムを採用

### 🏷 日本語名前自動分割（こだわり）

タブ移動時に苗字と名前の間に半角スペースを自動挿入：

- **JP_SURNAMES辞書**: 800件以上の日本語姓を搭載（随時拡充中）
- **最長一致アルゴリズム**: 「長谷川」と「長谷」が両方ある場合、正しく「長谷川」として認識
- **沖縄系姓対応**: 比嘉・金城・玉城・知念・宮城・与那嶺など
- **5文字姓対応**: 勅使河原・武者小路など
- 全角スペース・スペースなしどちらでも半角スペースに統一

### 📧 メール送信・バウンス検知（こだわり）

- **顧客・担当者への確認メール**: Gmail APIで自動送信。HTMLメールでサクピタロゴ入り
- **バウンス自動検知**: 送信後3秒待機し、GmailをmailerDaemon/postmasterで検索。バウンスを検知した場合は即アラートを表示
- **送信ステータスのカレンダー記録**: 成否をGoogleカレンダーイベントに自動PATCH
  - ✅ 送信成功: カレンダー説明欄に「確認メール送信済み」を記録
  - ⚠️ バウンス検知: イベントタイトルに「⚠️ メール未達」を追記、説明欄にも記録

### 📆 Googleカレンダーイベント作成

- **Google Meet自動生成**: 予約時にMeet URLを自動発行し、メール本文に記載
- **イベント説明欄のフォーマット**:

```
【会社名】株式会社〇〇
【顧客名】山田 太郎様
【部署名・役職名】営業部 部長
【電話番号】090-1234-5678
【メールアドレス】yamada@example.com
【コメント】（任意）

✅ 確認メール送信済み
送信先: yamada@example.com
```

### 📊 Googleスプレッドシート連携

- **自動作成**: 初回予約時にGoogleドライブへ「サクピタ 予約データ」シートを自動生成
- **列構成**: 企業名 / 担当者名 / 部署 / 役職 / 電話番号 / メールアドレス / メール送信日 / 商談日 / コメント
- シートIDはlocalStorageに保存し、次回以降は同じシートに追記

## 技術的こだわりポイント

### ⚡ 高速化・軽量化

- **Discovery Docs廃止**: `gapi.client.init`でdiscovery docsを読み込まず、`gapi.client.request()`の直接呼び出しに統一 → 初回ロード約600ms短縮
- **preconnect / dns-prefetch**: `<head>`にGoogleのAPIドメインへの事前接続ヒントを追加
- **busyDataキャッシュ**: 週単位・メンバー別にキャッシュし、同じ週のナビゲーション時はAPIを再呼び出しせず即表示
- **フレームワークなし**: React/Vue等を使わず純粋なVanilla JSで実装。バンドルサイズ最小化

### 🔄 リロード耐性

- アクセストークンをlocalStorageに保存し、`gapi.client.setToken()`で復元
- ページリロード後もサインイン画面を経由せずにカレンダーが即表示される
- 55分ごとのサイレント自動更新でセッションを半永続的に維持

### 🛡 バリデーション設計

- フロントエンドのみで完結する入力チェック（サーバー不要）
- エラー時は該当フィールドをハイライト + アラートで具体的なフォーマットを案内
- メールアドレスの正規表現チェックに加え、バウンス検知による二重確認
